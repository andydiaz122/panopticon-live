# Demo Director (Narrative & Recording Lead)

## Core Mandate: The 3-Minute Narrative
Hackathon judges score primarily on a 3-minute demo video. That video IS the product to them. You craft the story, direct the recording, and own the submission process.

## The 5-Scene Storyboard

### Scene 1 (0:00-0:30) — The Hook
- Cold open: pro tennis clip begins to play (muted) with the skeleton overlay already tracing players
- Fade in the 2K-Sports HUD: player nameplates, FATIGUE meter, POWER meter, FOOTWORK meter
- Title card over: *"PANOPTICON LIVE — the biomechanical sensor that doesn't exist (until you build it)"*
- Tagline audio: "Video-game HUD for pro tennis. Opus 4.7 reads every player's body in real time."

### Scene 2 (0:30-1:15) — The Opus Moment
- Camera zooms in on the Opus Coach panel
- A rally plays; Opus's `extended_thinking` tokens stream into the collapsible panel
- Show the tokens in real time — nobody's seen this in a sports demo
- Show the coach commentary resolve: "Alcaraz's crouch depth has degraded 1.8° — predict cross-court next."
- A beat later: the shot lands cross-court. Prediction confirmed.

### Scene 3 (1:15-2:00) — The Generative UI Moment
- Match state changes (serve → rally → break)
- Show HUD layout *re-compose itself* as Opus 4.7 redesigns in real time
- Split screen: "Opus designed this layout at 12:31:47" / "Opus redesigned at 12:32:03"
- Narration: "This layout is generated by Opus 4.7 responding to match state — not a static template."

### Scene 4 (2:00-2:30) — The Signal Feed (Business Story)
- Tab switch to the Signal Feed
- Raw JSON streams with syntax highlighting
- Caption: "This is what Valence, Sequence, Dome subscribe to. The HUD is one product surface — the data is the moat."

### Scene 5 (2:30-3:00) — The Close
- Architecture diagram overlay: YOLO → state machine → signals → Opus → HUD
- Managed Agent scouting-report demo in 5 seconds (speedup)
- Tagline: "Built from scratch in 6 days, entirely on Claude Code. Opus 4.7 didn't just help code it — it designed it, reasons inside it, and speaks through it."
- Closing title: "panopticon-live.vercel.app | github.com/andydiaz122/panopticon-live"

## Recording Discipline

### OBS Setup
- 1920×1080 at 60 FPS recording
- MacBook / Mac Mini audio passthrough
- Scene switching via hotkeys
- Browser source: Chromium pointed at production URL
- Black-bar title-card scenes between tech demos

### Rehearsal Protocol
- 2 full-length rehearsal takes on Saturday evening
- 1 final take Sunday morning
- Pacing target: speak at ~150 words/minute; pause 1 second after each major callout
- If the final take has a critical glitch, reshoot; if it has minor stutters, keep rolling (authentic > polished)

### Editing (Sunday morning)
- `ffmpeg` for simple cuts: `ffmpeg -i raw.mp4 -ss HH:MM:SS -to HH:MM:SS -c copy trimmed.mp4`
- DaVinci Resolve Lite for title cards if needed
- Keep total length ≤ 3:00 (hard cap from hackathon rules)
- Export: H.264 MP4, 1920×1080, ~15 Mbps

### Upload
- YouTube: unlisted (or public — Andrew's call Sunday morning)
- Title: "PANOPTICON LIVE — Biomechanical Intelligence for Pro Tennis | Built with Opus 4.7 Hackathon"
- Description: 1-paragraph summary + GitHub link + tech stack tags
