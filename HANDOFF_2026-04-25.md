## 4. Architecture Overview

### CV pipeline (offline precompute, runs on M4 Pro)
```
ffmpeg (stdout) → BytesIO → YOLO11m-Pose (MPS, conf=0.001, imgsz=1280)
  → PhysicalKalman2D (court-meter geometry, NOT screen-normalized coords)
  → RTS smoother (offline backward pass; 69.6% RMSE / 87.3% jitter reduction)
  → MatchStateMachine (PRE_SERVE_RITUAL / ACTIVE_RALLY / DEAD_TIME / etc.)
  → 7 signal extractors (BaseSignalExtractor ABC, symmetric tail handling)
  → DuckDB schema + Pydantic v2 contracts
  → match_data JSON written to dashboard/public/match_data/
```

### Dashboard (Vercel-deployed Next.js 16 + Turbopack + Bun)
- **Tab 1 — Live HUD**: video player + canvas skeleton overlay + biometric SignalBars + Tickertape (phase-weighted) + CoachPanel + PlayerNameplate
- **Tab 2 — Raw Telemetry**: dense monospace firehose (STATE / SIGNAL / INSIGHT / ANOMALY rows) + **"Download Match Data (.csv)" button** (the visible product artifact)
- **Tab 3 — Scouting Committee**: client-side replay of `agent_trace.json` showing the 3-agent Claude Managed-Agents swarm with timed pacing + FINAL TACTICAL BRIEF render

### Multi-agent swarm (offline pre-computed + replayed client-side per PATTERN-056 / DECISION-010)
3-agent orchestration: **Analytics Specialist → Technical Biomechanics Coach → Tactical Strategist**. The swarm runs offline during precompute via Claude Managed Agents, captures the full execution trace (thinking + tool_calls + handoffs) into `agent_trace.json`, then replays client-side with timed pacing. This immunizes the demo against Vercel's 10–15 s serverless timeout while still showing real multi-agent reasoning. Honest-disclosure banner: *"ARCHITECTURAL PREVIEW: SWARM ACCELERATED FOR DEMO."* Targets the "Opus 4.7 Use" judging criterion (25 %).

### Display-only G43 architecture (DECISION-016)
Authored content lives in `display_*` MatchData fields (`display_narrations`, `display_transitions`, `display_player_profile`); never merges with live telemetry. Provenance stamped `stubbed_mcp` in trace events. Solves the "three realities on one screen" drift between authored copy + live signals + Opus output.

### The new skill that drove the work

Created `/Users/andrew/Documents/Coding/hackathon-research/.claude/skills/remotion-cinematic-craft/SKILL.md` — the operationalized standard. Summary of contents:

- **Orthogonal Responsibility**: OWNS motion density, layered timing, typography animation, music sync, chapter narrative; DELEGATES color/font choice to `2k-sports-hud-aesthetic` and React patterns to `react-30fps-canvas-architecture`.
- **The 6 Failure Modes** (memorize, never repeat): (1) composition too short to tell a story; (2) single-element frames; (3) motion density too low; (4) no music = no rhythmic anchor; (5) typography choice without typography craft; (6) no chapter / narrative structure.
- **5 Anatomy Layers**: L1 background field, L2 framing chrome, L3 hero element, L4 supporting context, L5 secondary motion. A toy composition has only L3; world-class uses 4–5 layers, each with its own animation timing.
- **4 Typography Animation Patterns**: A — Kerning Sweep (letters from spaced to natural over 600 ms); B — Italic Reveal Beat (italic accent appears 200 ms after upright); C — Per-Character Stagger; D — Weight Modulation (variable font axis).
- **Music Sync Patterns**: pseudo-tempo 120 BPM = beat every 30 frames @ 60 fps = bar at 240 frames = 4 s. Place hard cuts and hero-slot swaps on bar lines so a future DaVinci audio bed automatically lands on downbeats.
- **10-Item Composition Checklist** (run BEFORE first render): duration ≥ 8 s, 3+ anatomy layers, motion density ≥ 3 events/sec, at least one typography animation pattern, pseudo-tempo timing, chapter identity, exit foreshadows next entry, tsc clean, ffmpeg frame extracts at 33/66/99 %, watch end-to-end at 1×.
